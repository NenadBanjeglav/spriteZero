# Email-Backed Vote With Controlled Location

Status: ready-for-agent

## What to build

Implement the first complete Email-Backed Vote path using Convex. A visitor should be able to submit an email and selected location from a temporary Controlled City Vote list, get a Counted Vote response, and receive a friendly Already-Counted Vote response when the normalized email already exists. Keep the vote flow low-friction with submission-based Vote Consent and Invisible Vote Protection.

## Acceptance criteria

- [x] Convex is configured for the app and documented in the local environment setup.
- [x] A vote mutation accepts email, location id, consent copy version, and honeypot data.
- [x] Emails are normalized before deduplication.
- [x] First-time submissions create a Counted Vote.
- [x] Duplicate normalized emails return an Already-Counted Vote success state instead of an error.
- [x] The vote form includes email, controlled location selection, hidden honeypot, and no required consent checkbox.
- [x] Public queries expose only aggregate vote totals and per-location counts, not raw emails.
- [x] Basic rate limiting or equivalent low-friction abuse protection is present.
- [x] Vote form labels, keyboard navigation, ESC close, and email autofill work.
