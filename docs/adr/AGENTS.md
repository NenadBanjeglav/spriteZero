# Purpose

- Own architecture decision records for stable campaign implementation choices.

# Ownership

- ADR files under this folder record accepted decisions that affect application architecture, services, assets, and deployment.

# Local Contracts

- Preserve the numbered ADR filename format: `NNNN-short-title.md`.
- New ADRs should state the decision, context, and trade-off clearly enough for future implementation work.

# Work Guidance

- Do not change an ADR to fit code after the fact; add or revise decisions only when the architecture decision itself changes.

# Verification

- Verify new ADR numbers are sequential and links from other docs resolve.
