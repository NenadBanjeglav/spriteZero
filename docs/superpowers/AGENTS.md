# Purpose

- Own Superpowers-generated design specs and implementation plans under `docs/superpowers/`.

# Ownership

- `specs/` owns approved brainstorming design documents that translate product scope into implementation contracts.
- Product scope remains in `.scratch/` PRDs and campaign vocabulary remains in root `CONTEXT.md`.

# Local Contracts

- Specs must cite the source PRD or issue path they refine.
- Keep implementation contracts aligned with accepted ADRs under `docs/adr/`.
- Use root `CONTEXT.md` terminology for campaign concepts.

# Work Guidance

- Write specs as durable handoff documents, not diary notes.
- When documenting current gaps, separate target-state requirements from existing implementation state.

# Verification

- Verify referenced local paths exist before closing documentation work.
